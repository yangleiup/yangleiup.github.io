Webpack 提供打包功能与一套文件处理机制，通过生态的各种Loader与Plugin对代码进行预编译与打包 常规配置 Entry 入口文件 Output 出口路径 Module 模块,webpack中任何文件都可以做一个模块，可以根据配置不同的Loader进行加载与打包 Chunk:代码块,可以根据配置，将所有模块代码合并成一个或多个代码块，按需加载，提高性能 Loader 模块加载器，进行各种文件类型的加载与转换 Plugin 扩展插件，可以通过Webpack相应的事件钩子，介入到打包过程中的任意环节，对代码按需修改 工作流程 读取配置文件，按命令初始化配置参数，创建Compiler对象 调用插件的apply方法挂载插件建通，然后从入口文件开始执行编译 按文件类型，调用相应的Loader对模块进行编译，触发各个时间点的时间，调用Plugin执行，再根据模块依赖查找到依赖模块，递归执行这里的流程。 将编译后的代码包装成一个个代码块，按依赖于配置确定输出内容，然后通过Plugin进行文件修改 最后根据Output把文件内容一一写入到指定的文件夹中 Loader 对webpack传入的字符串进行按需修改, 即插即用 链式传递，基于NODE环境，有文件增删的较高权限，可同步也可异步 Babel-loader babylon将ES6/7解析成AST babel-traverse对AST进行遍历转译，得到新的AST 新的AST通过babel-generator转换成ES5 常用Loader file-loader 加载文件资源,移动/复制/命名等 url-loader 常用于加载图片，可将小图片转换成DATA Url babel-loader ts-loader, 编译TypeScript style-loader 将css代码以style标签形式插入到HTML中 css-loader 分析@import和 url(), 引用css文件 postcss-loader 用于css的兼容性处理, 具有众多功能，如添加前缀，单位转换等 less-loader / sass-loader : css预处理器 Plugin Plugin可以监听webpack的事件，在相应时间点对打包内容进行修改 事件流机制,让 Plugin可以干预整个生产过程中的每个步骤 以观察者模式的的实现事件的订阅与广播的核心基础类Tapable 常见Plugin UglifyJs 压缩混淆 CommonChunk 代码分割 ProvidePlugin 自动加载模块 html-webpack-plugin 加载HTM文件，并引入css/js文件 extract-text-webapck-plugin / mini-css-extract-plugin 抽离样式,生成css文件 DefinePlugin 定义全局变量 optimize-css-assets-webpack-plugin css代码去重 webpack-bundle-analyzer 代码分析 compression-webpack-plugin 使用gzip压缩js 和css happypack 使用多进程，加速代码构建 EnvironmentPlugin 定义环境变量 DLLPlugin,DLLReferencePlugin 提前进行打包与缓存，比米娜每次重新编译 重要类 Compiler 相当于Webpack实例，包含当前webpack的所有配置信息，在启动时完成创建，随生命周期传递 Compilation 编译实例，当文件发生改变，webpack就会创建一个新的Compulation对象，开始新的编译。通过它提供的API，可以建通每次编译过程中触发的事件钩子 常见优化 无用代码删除 如UglifyJS 摇树优化,消除被引用但未被使用的模块代码 代码分割 懒加载与异步加载 可使用SplitChunksPlugin, 按页面/功能/文件修改频率拆分 scope hoisting作用域提升 分散的模块划分到同一作用域中,必满代码的重复引用 webpack编译优化 热替换 缩小编译范围 modules 指定模块路径，减少递归搜索 mainFields 指定入口文件描述字段，减少搜索 noParse 避免对非模块化文件的加载 includes/exclude 指定搜索范文/排除 alias 缓存目录，避免重复寻址 多进程并发 webpack-parallel-uglify-plugin 多进程并发压缩js HappyPack 多进程并发文件的Loader解析 第三方模块缓存 DLLPlugin DLLReferencePlugin 打包后分析 Webpack Analyse/webpack-bundle-analyzer 对打包后文件进行分析
